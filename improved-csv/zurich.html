Zurich CSV Data

Files sizes for each day range from 16 to 25 megabytes.

The fields included in each .CSV include the following fields, in order of appearance:

  • tripStart
  • routeNumber
  • runNumber
  • direction
  • stopSequenceNr
  • stopId
  • stopNameShort
  • arrivalScheduled
  • departureScheduled
  • arrivalActual
  • departureActual
  • vehicleId
  • stopName
  • latitude
  • longitude
  • passengersBoardingTrip
  • passengersAlightingTrip
  • passengersDifferenceTrip
  • passengersBoardingStop
  • passengersAlightingStop
  • passengerLoadStop

The files are sorted according to the 'arrivalActual' field.

Sime records are missing scheduled arrivals. This was true in the source file.

The JavaScript procedures used to create the files included:

Open main file, 50 gigs at a time - the max JavaScript seemed to allow and save into 14 smaller files.

Join the two half day files back into one file.

Sort each day file in the order of actuall arrival time

Open stops.json. Read its name and latitude and longitude data. Add this data to each record in each day file.

Open the file with the passenger load data and add that data to each day file. Allow for the the passenger load file having data for perhaps only twenty percent of the main file records. Also figure out which records belong to each other.
