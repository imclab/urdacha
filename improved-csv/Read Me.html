The data files provided by the organizers included two very large .CSV files.

Extracting and parsing the data from a CSV file with, for example, 526,364,977 bytes (Zurich) or even 182,578,664 bytes(Geneva's schedule-real-time.csv) took some thinking. Applications such Open Office and Notepad++ were found to be simply incapable of dealing with files of this saze.

Suggestion to organizers: If you ever work again on a project like this with very large data sets, consider offering a files under 100 megabytes in size along side the very large single data sets.

General goals for the generated data are generally described on the Urdacha Wiki.

The Urdacha team is pursuing two quite different approaches towards opening and parsing the files.

The first approach uses PostgreSQL and a variety of RDBMS tools to parse, extract and reformulate the data. The generated CSV files are available via DropBox here:

Urdacha

The link also provides a variety of tools as well as the useful document:'Steps to create SFMTA Files'.

The second approach uses JavaScript and the XMLHttpRequest feature to load the large files in smaller portions. Once loaded the files are parsed and saved locally using the new FileSaver

All the files produced using this technique are available on the GitHub Urdache repository 'improved-csv'. The folders for Geneva and Zurich also include all the JavaScript tools that are being use to manipulate the files.
